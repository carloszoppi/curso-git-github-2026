COMANDOS BASH

ls -> lista arquivos
cd -> move entre diretórios
pwd -> mostra diretório de trabalho - caminho absoluto onde estou
cat -> exiber conteúdo do arquivo
rm -> remove arquivo ou diretório
rm -rf -> remove de forma recursiva e forçada todos os diretórios

desktop - significa área de trabalho

Toda pasta tem dois arquivos dentro:um chamado . e outro chamado ..
Se de ls estes arquivos não aparecem. Se digitar ls -a vão aparecer arquivos
ocultos ./ e ../
Arquivos ocultos são os que começam com ponto. O arquivo ../ é a pasta mãe
da pasta que você está (agora estamos na pasta de trabalho - Desktop). 
Se você não sabe qual é a pasta acima dela é só digitar
cd ..
Para voltar para a pasta Desktop:
cd Desktop
O ./ é a própria pasta, é ela apontando para ela mesma. É só uma referência
é a própria pasta.

Criar a pasta nova: 
mkdir nomedapasta  - cria uma nova pasta no diretório que você está.
Para entrar na pasta: 
cd nomedapasta
Para saber o que tem:
ls -a (aparece os arquivos mais o ocultos ./ e ../
Para voltar para Desktop (o diretório acima)
cd ..  (ou seja, volta uma pasta acima)

Para completar a digitação, por exemplo, Desk e aperta TAB que ele completa
Desktop (o TAB fica embaixo do 1).

O "CTR L" dá uma limpada na tela.


Comando git init .   torna a pasta especial, ou seja, torna a pasta possível
de controlar versionamento.

Carlos@DESKTOP-VM1IU3C MINGW64 ~/Desktop/curso_git_2026
$ git init .
Initialized empty Git repository in C:/Users/Carlos/Desktop/curso_git_2026/.git/

Vamos entender - ele está dizendo que foi iniciado um arquivo de 
repositório vazio no endereço citado.
O .git/ não aparece porque é arquivo oculto. O que torna esta pasta epecial 
é a pasta dentro dela que é o . git/

Se der um rm - rf .git/ a pasta é apagada e volta a ser o que era antes.
